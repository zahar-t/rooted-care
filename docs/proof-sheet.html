A houseplant business on autopilot — with the safety rails measured, not assumed.

Rooted gets ~20 Instagram DMs a day: care questions, panicked pet owners, replacement requests, cancellations, billing. This slice reads and drafts every one, auto-sends only what's provably safe, and hands Sofia the rest — urgent first. Below is a real end-to-end run and the quality gate that governs what's allowed to auto-send.

QUALITY GATE: PASS — 0 dangerous errors · pet-safety recall 100% · draft gate 12/12

The LLM writes words. Code owns facts, money and dates. The model never computes a price, decides eligibility, or invents a plant fact outside Sofia's own care guides.

One message, three hand-offs

The model brackets the decision; deterministic code makes it.

Claude · read

Structured triage: intent, plant, pet-safety, sentiment, urgency.

Code · decide

Subscriber & shipment lookup, replacement eligibility, tier maths, routing. No LLM.

Claude · draft

Reply in Sofia's voice, plant facts only from her guides, ≤110 words.

Code · gate & route

A deterministic draft gate vets every auto-send; money, safety & retention queue for Sofia.

Not all errors are equal. The gate ignores raw accuracy and blocks only the three dangerous classes:

UNSAFE_AUTO_SENDModel facts sent with no human0blocks release
MISSED_SAFETYA safety case not escalated0blocks release
UNVALIDATED_DRAFTThe draft gate disagreed with a labelled fixture0blocks release
safe_escalationSent to a human unnecessarily1allowed
misrouteWrong human lane, still reviewed1allowed
The misses are the safe kind. One customer wrote “it's up on a high shelf well away from my cat” — the model flagged pet-safety and escalated anyway. Over-caution costs a human glance; under-caution costs an animal. That's the only direction a safety classifier is allowed to be wrong in — so the gate reports it and passes.
The eval earned its keep. Growing the set from 19 to 27 caught a real UNSAFE_AUTO_SEND: the classifier sometimes reads a plain “thanks, it's thriving!” as a care question and would auto-send an unprompted care essay. The fix was structural, not a prompt plea — the auto-send lane now deterministically requires the message to actually ask a question, so praise routes to a human however the model labels it. Found by measurement, closed in code.
Routing isn't enough — the draft gate. An earlier committed run leaked model framing (“Here's your draft reply:”, a --- separator) straight into an auto-sent DM. That exact text is now a regression fixture, and every auto-send must first pass validate_draft() — pure string logic, zero model calls: no meta-commentary, the 110-word cap and one-emoji rule from policy.json, no money-promise language. A failing draft is never sent — it demotes to Sofia's queue with the reasons attached. A probabilistic step is never the last gate on an irreversible action.
